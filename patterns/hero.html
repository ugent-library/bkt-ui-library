Hero

The bt-hero component is the full-bleed introductory section used on the public homepage and research detail pages. It stacks a decorative SVG background layer behind a constrained content column.

Structure

Three parts: the shell (.bt-hero), the absolute background (.bt-hero__bg), and the content column (.bt-hero__content). The background layer is pointer-events: none and purely decorative.

Research detail hero

On research detail pages the hero carries the record title, badges, and primary actions. The SVG globe animation from _svg-animations.scss runs in the background. The hero is still data-surface="public" even when reached from the backoffice — it is always the public-facing record view.

SVG animation classes

The SVG uses CSS classes defined in _svg-animations.scss — not inline styles or fill/stroke attributes. This lets the animation respond to the surface token --bg (which maps to --bt-gray-25) and the ink-alpha tokens --i02–--i40.

svg class reference
<!-- Background fill -->
<rect fill="var(--bg)">           <!-- bt-gray-25 -->

<!-- Strokes -->
<path class="gridLine">           <!-- faint grid -->
<path class="stroke noFill edge muted">  <!-- muted path -->
<path class="stroke noFill edge mid">    <!-- slightly more visible path -->

<!-- Fills -->
<circle class="fillFaint pulse">  <!-- faint node, animated opacity -->
<circle class="fillMuted pulse">  <!-- muted node -->
<circle class="fill">             <!-- solid primary ink -->

<!-- Animation helpers -->
<circle class="particle">         <!-- moving dot along a path -->
<g class="frontSoft">             <!-- front layer with soft opacity -->
