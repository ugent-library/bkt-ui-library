Search & filtering

How the search box, suggest panel, sidebar facets, Add-filter picker, chip bar, and results toolbar compose into one system. This page shows the assembly; the interaction rules — URL-is-truth, what the box exposes, where each dimension goes — live in docs/SEARCH-AND-FILTERING.md. Each part has its own kit page for markup and states.

The parts and their jobs

Six components, two jobs. The box enters a result space or navigates to a record; the sidebar and picker narrow within it; the chip bar reads out what is applied. The URL holds the one canonical state — every part is a view onto it.

Part Job Documented in
Search box Enters a result space (?q=) or jumps to a record. It doesn't filter the list you're viewing — narrowing is the sidebar's and picker's job. Search bar
Suggest panel Autocomplete of navigation shortcuts — entities you can go to, plus keyword/tag topic-sets. Search bar
Sidebar facets Narrows on browse-it dimensions — closed, low-cardinality, countable (Access, Type, Year, Language). Facets
Add-filter picker + editor Narrows on search-it dimensions — high-cardinality records (author, journal, project, organisation) via typeahead. Filter picker
Chip bar Reflects the applied filters as editable split chips — a readout of the sidebar/picker, not an input. Filter picker, Badges
Results toolbar Presentation controls — result count (live region), sort, page size, and result actions. Toolbar

Not part of this system: bt-sidebar (Sidebar navigation) is the persistent nav column in u-layout--app — a navigator, not a filter rail. The filter rail is a plain <aside>.

Assembled — a public results page

The parts in place on public-works: search box on top, the chip bar (readout) and Add-filter picker (input) below it, the facet rail on the left, and the results toolbar above the list.
