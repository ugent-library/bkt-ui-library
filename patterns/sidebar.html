Sidebar Navigation

bt-sidebar is the persistent left-column navigator inside u-layout--app. Use it for real page navigation only — not as a filter panel or decorative element.

Basic structure

Links use Bootstrap .nav-link directly on <a> elements inside the sidebar. The SCSS provides flex layout, gap, and padding — no Bootstrap utility classes needed on the link. Group labels use .bt-sidebar__group-label. Wrap link text in .bt-sidebar__label so it hides cleanly in slim mode.

Badges

Drop a Bootstrap badge directly inside the link. The SCSS pushes it to the right automatically via margin-left: auto — no ms-auto needed. Add aria-hidden="true" on the badge and put the count and context in aria-label on the link. Badges are hidden in slim mode automatically.

Badge colour signals urgency: bg-danger for overdue/errors, bg-warning for items needing attention, bg-primary for neutral counts.

Collapsible — slim mode

Adding .bt-sidebar--slim collapses the sidebar to icon-only width (3rem). Group labels, link text, and badges are hidden via CSS. The toggle button updates aria-expanded and aria-label — handled by sidebar-toggle.js, which also initialises Bootstrap tooltips on all links at page load.

Expanded — click the arrow to collapse

Layout integration

bt-sidebar is a direct child of u-layout--app placed in the sidebar grid area. The shell handles sticky positioning and scroll — do not add position: sticky or overflow rules to the sidebar element.

Sidebar width is controlled by two CSS custom properties on u-layout--app: --u-nav-sidebar-width (default 13rem) and --u-nav-sidebar-width-slim (default 3rem). Override them on the shell element if a page needs different widths.

Available classes

Class Element Purpose
bt-sidebar <nav> Base sidebar container. Sets width, padding, and scroll behaviour.
bt-sidebar--bordered Modifier on bt-sidebar Adds a right-edge border. Use for detail-page section navs, not the global app sidebar.
bt-sidebar--slim Modifier on bt-sidebar Collapses to icon-only width (3rem). Hides group labels, link text, and badges.
bt-sidebar--flush Modifier on bt-sidebar Removes horizontal padding. Use when links provide their own internal spacing.
bt-sidebar__toggle <button> Collapse/expand trigger. Toggles bt-sidebar--slim on the parent nav.
bt-sidebar__group-label <p> Section heading inside the sidebar. Hidden automatically in slim mode.
bt-sidebar__label <span> inside .nav-link Link text. Must be wrapped in this span so slim mode can hide it while keeping the icon visible.

Bootstrap recommendations

The sidebar uses Bootstrap's .nav-link on <a> elements directly. The SCSS scopes all flex layout, spacing, and badge alignment inside .bt-sidebar — no Bootstrap utility classes needed on the links themselves.

Active state

Set both aria-current="page" and class="nav-link active" on the active link. aria-current is what screen readers use; .active handles the visual background. In server-rendered templates, set both explicitly.

Tooltips

Add data-bs-toggle="tooltip" data-bs-placement="right" title="…" to every link. Bootstrap reads the title attribute — no extra markup needed. sidebar-toggle.js initialises all tooltips at page load by querying .bt-sidebar a.nav-link[data-bs-toggle="tooltip"]. Tooltips are present at all times but become essential when the label is hidden in slim mode.

Mobile — consider Offcanvas

Bootstrap's Offcanvas handles mobile drawers that slide over content. If the sidebar needs to become a drawer on small screens, wrap it in an Offcanvas body rather than building a custom mobile variant. bt-sidebar styles remain usable inside the Offcanvas body with no changes.
