Facets

Filter sidebars are plain Bootstrap: fieldset/legend groups with form-check rows, separated by <hr>. No custom facet classes. Used on the public search sidebar today (public-works). Backoffice list/search facets are a separate surface that is not yet ready — so every example here is a public facet.

The facet sidebar in the search system

The facet sidebar is one input in the composed search system — see Search & filtering for the assembly and docs/SEARCH-AND-FILTERING.md for the interaction rules. Three things carry over into how you build it:

  • Readout, not just input. Ticking a box surfaces a removable chip; removing that chip unticks the box. Both are views onto the URL (server-derived in production). The chips themselves are the Filter picker's split-chip markup.
  • How facets combine. Multiple values in one facet are OR; different facets are AND. There is no boolean control on the public surface — operator choice is an advanced/expert affordance.
  • The public set is deliberate. Public carries only the browse-it dimensions — Access, Type, Year, Language, with Access first (OA-forward). High-cardinality dimensions (author, journal, project, organisation) go to the Add-filter picker, not the rail. The full placement taxonomy is in the doc (Rule 5).

The rail is a plain <aside aria-label="Filter by"> — not bt-sidebar, which is the app navigation column.
