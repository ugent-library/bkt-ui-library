Deposit Components

Components specific to the deposit (add publication) workflow: publication type picker, file upload area, and author/contributor rows. These live inside u-layout--deposit and always run on the backoffice surface.

bt-radio-card — publication type picker

A grid of large radio inputs styled as selectable cards. The hidden <input type="radio"> drives state; the visible .bt-radio-card__body responds via adjacent sibling CSS. Group them in .bt-radio-card__group for the responsive grid layout.

Standard — one type selected

Select publication type

bt-file-drop — file upload area

An accessible drag-and-drop zone. The entire area is a <label> that activates a visually-hidden <input type="file">. Drag-over state is added via JavaScript by toggling the class drag-over.

Default — idle state

Drag-over state — .drag-over toggled by JS

Uploaded file row

Once a file is attached, it is listed above the drop zone as a simple flex row with Bootstrap utilities. No dedicated component — the structure is just d-flex align-items-center gap-3 p-3 bg-white border rounded.
