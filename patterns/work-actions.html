Work actions

The action rows on research output — search result cards and the detail page header. v1 keeps file access on the page itself; v2 adds direct access CTAs. Rules: see raven#141.

One primary per context: the access CTA. Add to list is btn-secondary on the detail header, btn-ghost on cards; Cite is btn-ghost everywhere. A CTA that navigates (login, external, file) is an <a>, never a <button>.

Search result card — v1

Every card: Cite (ghost) + Add to list (ghost). No access CTA — the title navigates, the badge communicates access. The Add to list picker panel is documented at Panel.

Search result card — v2

The card gains the access CTA, mirroring the detail header with a compressed label.
