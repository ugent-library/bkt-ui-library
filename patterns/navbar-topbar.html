Navbar & Topbar

The bt-navbar component is the persistent application topbar. It appears on every page, sticks to the top, and defines the primary navigation and identity of the surface. Backoffice and public variants differ in content and tone — not in structure.

Backoffice navbar

Carries the application logo, section navigation, role switcher, and account menu. The account avatar doubles as a dropdown trigger.

Public navbar

Carries the repository identity, site navigation, and a shortcut to the researcher's own deposit workflow. The site name acts as the <h1> equivalent for identity — it is a heading styled as nav text, not a bare link.

Active link state

Apply aria-current="page" to the link matching the current URL. The CSS responds to this attribute — no extra class needed.

Nav with active item

Research output Datasets People
html
<!-- aria-current="page" drives the active style — no extra class -->
<a href="/research" class="bt-navbar__link" aria-current="page">Research output</a>
<a href="/datasets" class="bt-navbar__link">Datasets</a>

Skip link

Every page must include a skip link before the navbar as the first focusable element. It is visually hidden until focused.

html
<!-- First element in <body> — before bt-navbar -->
<a href="#main-content" class="skip-link">Skip to main content</a>

<header class="bt-navbar" role="banner">…</header>

<main id="main-content">…</main>

Layout integration

The navbar is always a direct child of a layout shell. It fills the navbar grid area and sticks to the top via position: sticky; top: 0. The --s-topbar-height token (52px) is used by sidebar children to offset their own top value.

html — u-layout--app
<div class="u-layout--app" data-surface="backoffice">
  <header class="bt-navbar" role="banner">…</header>
  <nav class="bt-sub-sidebar" aria-label="Section navigation">…</nav>
  <main id="main-content">…</main>
</div>
