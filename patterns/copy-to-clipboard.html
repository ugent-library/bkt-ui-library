Copy to clipboard

A value shown in a <code> next to a button that copies it. One behaviour for both variants below, so what is displayed is always what gets copied.

Contract

Put data-clipboard on the button and a <code> next to it in the same parent. On click, assets/js/clipboard.js copies that <code>'s text and confirms for two seconds — the icon becomes a check, and a button with a .btn-text label also swaps to “Copied!”. Because the copied text is read from the visible <code>, there is no second value to fall out of sync. Button and <code> may sit in either order.

No inline onclick: the behaviour is the shared script, not markup. The button variant itself is documented on Buttons.

Copy Biblio ID

A labelled action beside a record's Biblio ID, in the work-card footer and across the backoffice. The visible button text is the accessible name; the icon is aria-hidden.

Persistent link

The “Link to this record” box on public detail pages (work, dataset, person, organisation, project). It shows the record's persistent URL — a handle for works, the canonical page for the rest — and copies exactly that. The copy button is icon-only, so its accessible name lives in aria-label; the script restores that label after confirming.
