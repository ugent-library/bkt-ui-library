People search

An inline search that resolves a person from the PersonIdentity index. Used in the author filter editor and the deposit flow add-author form. Not a modal — the results list appears directly below the input.

Empty — before typing

The input is focused. No results list yet. The hint below the input explains what the field searches across.

Type a name to search across UGent people and external authors.

Results — query "doe"

Results appear in a bordered list below the input. The query term is highlighted in the name with <mark>. Each row shows the person's affiliation, department, active years, and ORCID when available. UGent people show their UGent ID. External people show "External" as affiliation. Keyboard-navigable: Arrow keys move between rows, Enter selects.

Selected

After the user clicks a row, the list collapses to show only the selected person. The row gets .is-selected — blue background, checkmark on the right. The input value updates to the person's name. The user can clear the selection to search again.

No results

When the query returns nothing, the list shows a plain text message. No illustration, no secondary action — the user should refine their query or use "Add as external author" if available in the context.

In context — author filter editor

People search is always hosted inside a .filter-editor panel. The panel provides the title, the action row (Apply / Cancel / Remove filter), and constrains the width. The search input and results list fill the available space between them.

Search input + results list

<input type="search" id="people-search-input" class="form-control form-control-sm"
  placeholder="Search by name or ORCID…"
  autocomplete="off"
  hx-get="/people/search"
  hx-trigger="keyup changed delay:300ms"
  hx-target="#people-results-list"
  hx-indicator="#people-search-indicator">

  Searching…

  aria-label="People matching your search" hidden>
  <!-- populated by HTMX → people-search-results.html -->

Classes

All classes are defined in assets/scss/patterns/_booktower-components.scss. No new SCSS is needed to use this pattern.

Class Element Purpose
.people-results div[role="listbox"] Scroll container for result rows. Bordered, rounded, max-height with overflow-y auto.
.people-result div[role="option"] A single person row. Flex, cursor pointer, border-bottom between rows.
.people-result.is-selected — Blue background. Add aria-selected="true" at the same time.
.people-result__icon span Icon column, flex-shrink-0, muted colour.
.people-result__name div Person's display name. Contains <mark> for query highlights.
.people-result__meta div Flex row of metadata items, xs text, muted colour.
.people-result__meta-item span One metadata token (affiliation, dept, years, ORCID). Inline-flex with icon.

Behaviour

  • Search fires after 2 characters with a 300ms debounce.
  • In production: hx-get="/people/search?q=" returning the people-search-results.html partial.
  • In the prototype: the JS array PEOPLE in filter-editor.html stands in for the server.
  • Clicking a row updates the input value, collapses the list to the selected state, and stores the person's data-id and data-name for the filter chip or form submission.
  • The if-building icon used for department is not in the verified icon list — use if-building as a fallback until confirmed.
  • An unlinked (external) author is valid. The no-results state should lead to the "Add as external author" fallback if present in the host context (deposit form), not block submission.
