Patterns

Layout Shells

Two layout shells handle every page in the system. Each shell owns its topbar and main content area. The backoffice shell can host the navigation sidebar and optional internal panes around the same main structure.

The two shells

All shells use --s-topbar-height (52px, defined in _surfaces.scss) for sticky sidebar offsets. Public <main> uses Bootstrap .container inside it — the shell itself has no max-width. The app shell can also host an optional right-side u-main__panel for contextual actions.

u-layout--app — backoffice list/detail pages

Structure

bt-navbar and bt-sidebar are shell regions. Inside main, use u-main__header, u-main__body, and optional u-main__footer.

When the body splits, u-main__sidebar scrolls on the left, while u-main__content-body scrolls inside the right pane between an optional pane header and pane footer.

If you need contextual actions next to the page, add an optional shell-level u-main__panel as a sibling of main.

html
<div class="u-layout--app" data-surface="backoffice">
  <header class="bt-navbar" role="banner">…</header>
  <nav class="bt-sidebar" aria-label="Section navigation">…</nav>
  <main id="main-content">
    <div class="u-main__header">…</div>
    <div class="u-main__body">
      <aside class="u-main__sidebar">…</aside>
      <section class="u-main__content">
        <div class="u-main__content-header">…</div>
        <div class="u-main__content-body">…</div>
        <div class="u-main__content-footer">…</div>
      </section>
    </div>
    <div class="u-main__footer">…</div>
  </main>
  <aside class="u-main__panel" aria-label="Actions">…</aside>
</div>

u-layout--public — public search and record pages

Structure

Public pages use the same u-main__header, u-main__body, and optional u-main__footer blocks inside main.

The difference is behavioral: public layouts use normal document scrolling, and content width comes from Bootstrap .container elements inside those blocks.

html
<div class="u-layout--public" data-surface="public">
  <header class="bt-navbar" role="banner">…</header>
  <main id="main-content">
    <div class="u-main__header">
      <div class="container">…</div>
    </div>
    <div class="u-main__body">
      <div class="container">…</div>
    </div>
  </main>
</div>

Topbar height token

--s-topbar-height is set in _surfaces.scss. If the topbar height ever changes, update the token in one place.

scss — _surfaces.scss
/* In _layouts.scss — no fallback needed, token is always defined */
.u-layout--app > .bt-sidebar {
  top: var(--s-topbar-height);
  height: calc(100vh - var(--s-topbar-height));
}

Internal layout — backoffice list pages

Inside u-layout--app > main, backoffice list pages follow a consistent vertical stacking order. Each piece is a separate component class.

Stacking order

.u-main__header optional fixed page header
.u-main__sidebar optional left pane
.u-main__content-header optional right-pane header
.u-main__content-body required scrolling pane body
.u-main__content-footer optional right-pane footer
.u-main__footer optional fixed page footer
html — inside u-layout--app main
<main id="main-content">
  <div class="u-main__header">...</div>
  <div class="u-main__body">
    <aside class="u-main__sidebar">...</aside>
    <section class="u-main__content">
      <div class="u-main__content-header">...</div>
      <div class="u-main__content-body">...</div>
      <div class="u-main__content-footer">...</div>
    </section>
  </div>
  <div class="u-main__footer">...</div>
</main>
