Patterns

Layout Shells

Two layout shells handle every page in the system. Each shell owns its topbar and main content area. The backoffice shell can host the navigation sidebar and optional internal panes around the same main structure.

The two shells

All shells use --s-topbar-height (52px, defined in _surfaces.scss) for sticky sidebar offsets. Public pages use normal document flow with Bootstrap containers inside main. App pages use a fixed shell with independent scrolling regions and can optionally host a shell-level u-main__panel.

u-layout--public

Use this for public search, detail and landing pages.

The shell is intentionally light: topbar plus normal flowing main.

Width and spacing come from Bootstrap .container, .row, and utility classes inside the layout regions.

u-layout--app

Use this for backoffice list, detail and workflow pages.

The shell owns sticky regions and scroll behavior.

Keep u-main__body plain by default. Add u-main__body--split only when the body contains a left u-main__sidebar.

Public Layout

Public pages reuse the same region names, but the page scrolls normally and content width comes from Bootstrap containers inside those regions.

html
<div class="u-layout--public" data-surface="public">
  <header class="bt-navbar" role="banner">...</header>

  <main id="main-content">
    <div class="u-main__header">
      <div class="container">...</div>
    </div>

    <div class="u-main__body">
      <div class="container">...</div>
    </div>

    <div class="u-main__footer">
      <div class="container">...</div>
    </div>
  </main>
</div>

Public Rules

Do not use u-main__body--split unless the page truly has an internal pane layout.

Prefer Bootstrap layout primitives inside u-main__header, u-main__body, and u-main__footer.

App Layout

This is the canonical app-shell markup for split backoffice pages. Remove optional blocks rather than inventing parallel variants.

html
<div class="u-layout--app" data-surface="backoffice">
  <header class="bt-navbar" role="banner">...</header>
  <nav class="bt-sidebar" aria-label="Section navigation">...</nav>

  <main id="main-content">
    <div class="u-main__header">...</div>

    <div class="u-main__body u-main__body--split">
      <aside class="u-main__sidebar u-main__sidebar--border-right">...</aside>

      <section class="u-main__content">
        <div class="u-main__content-header">...</div>
        <div class="u-main__content-body">...</div>
        <div class="u-main__content-footer">...</div>
      </section>
    </div>

    <div class="u-main__footer">...</div>
  </main>

  <aside class="u-main__panel" aria-label="Contextual actions">...</aside>
</div>

Page Regions

u-main__header, u-main__body, and u-main__footer are the page-level regions inside main.

Split Body

u-main__sidebar and u-main__content belong inside u-main__body u-main__body--split.

Right Pane

u-main__content-body is the scrolling pane. u-main__content-header and u-main__content-footer stay fixed around it.

Topbar height token

--s-topbar-height is set in _surfaces.scss. If the topbar height ever changes, update the token in one place.

scss — _surfaces.scss
/* In _layouts.scss — no fallback needed, token is always defined */
.u-layout--app > .bt-sidebar {
  top: var(--s-topbar-height);
  height: calc(100vh - var(--s-topbar-height));
}

App Anatomy

Inside u-layout--app > main, backoffice pages follow a consistent vertical stacking order. Use this to reason about what scrolls and what stays fixed.

Stacking order

.u-main__header optional fixed page header
.u-main__sidebar optional left pane
.u-main__content-header optional right-pane header
.u-main__content-body required scrolling pane body
.u-main__content-footer optional right-pane footer
.u-main__footer optional fixed page footer
html — inside u-layout--app main
<main id="main-content">
  <div class="u-main__header">...</div>
  <div class="u-main__body u-main__body--split">
    <aside class="u-main__sidebar">...</aside>
    <section class="u-main__content">
      <div class="u-main__content-header">...</div>
      <div class="u-main__content-body">...</div>
      <div class="u-main__content-footer">...</div>
    </section>
  </div>
  <div class="u-main__footer">...</div>
</main>

Variants

Single-Column Body

Use a plain u-main__body when the page does not need an internal left pane.

<div class="u-main__body">
  <section class="u-main__content">
    <div class="u-main__content-body">...</div>
  </section>
</div>

Split Body

Use u-main__body--split only when a left body sidebar is present.

<div class="u-main__body u-main__body--split">
  <aside class="u-main__sidebar">...</aside>
  <section class="u-main__content">
    <div class="u-main__content-body">...</div>
  </section>
</div>
