Patterns

Layout Shells

Three CSS grid shells handle every page layout in the system. Each shell owns its topbar, optional sidebar, and main content area. They replace the old flex + overflow: hidden approach that broke sticky positioning.

The three shells

All shells use --s-topbar-height (52px, defined in _surfaces.scss) for sticky sidebar offsets. Public <main> uses Bootstrap .container inside it — the shell itself has no max-width.

Topbar height token

--s-topbar-height is set to 52px in _surfaces.scss. The layout shells use it directly — no fallback. If the topbar height ever changes, update the token in one place.

scss — _surfaces.scss
:root {
  --s-topbar-height: 52px;
}

/* In _layouts.scss — no fallback needed, token is always defined */
.u-layout--app > .bt-sub-sidebar {
  top: var(--s-topbar-height);
  height: calc(100vh - var(--s-topbar-height));
}

Internal layout — backoffice list pages

Inside u-layout--app > main, backoffice list pages follow a consistent vertical stacking order. Each piece is a separate component class.

Stacking order

bt-toolbar — page title + primary actions
bt-filter-bar — search input + active filter tags
bt-bulk-bar — bulk actions (hidden unless rows selected)
bt-facets-col
bt-results-col → bt-results-toolbar + results + bt-pagination-bar
html — inside u-layout--app main
<main id="main-content">

  <div class="bt-toolbar">
    <div class="bt-toolbar__left"><h1 class="bt-toolbar__title">…</h1></div>
    <div class="bt-toolbar__right"><!-- actions --></div>
  </div>

  <div class="bt-filter-bar" role="search" aria-label="Search and filter">
    <!-- search input + filter tags + htmx-indicator -->
  </div>

  <div class="bt-bulk-bar" hidden>
    <!-- bulk action buttons, shown when rows selected -->
  </div>

  <div class="bt-content-area">
    <aside class="bt-facets-col" aria-label="Filter by">
      <div class="bt-facets">…</div>
    </aside>
    <div class="bt-results-col">
      <div class="bt-results-toolbar">…</div>
      <!-- results: cards or table -->
    </div>
  </div>

</main>
