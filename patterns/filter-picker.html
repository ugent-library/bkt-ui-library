Filter picker & editor

One connected interaction: a grouped filter menu (the picker) and an inline editor panel that opens when a filter is chosen, with applied filters shown as chips. Live on the public works page and the researcher / project directories, all driven by one engine — filter-bar.js.

How it works

The full flow, end to end:

  1. User clicks Add filter → Bootstrap dropdown opens showing the picker
  2. User clicks a filter item (e.g. Faculty or department) → dropdown closes, editor panel appears over the result list
  3. User sets a value → clicks Apply → editor closes, a chip appears in the active filter bar
  4. User clicks a chip → editor reopens pre-populated for editing
  5. User clicks × on a chip → filter removed

Wired by filter-bar.js — one engine with a config per bar (works, researchers, projects), self-discovered by the bar's id prefix. Chips are client-side only in the prototype.

Picker

A short, flat list of [data-filter] items in a bt-dropdown-scroll — no search input, because each filter set is small enough to scan. The researcher directory offers Faculty/department, membership, and public-output; the project directory swaps in host faculty, status, and a period range; public works uses the search-it set — Author, Organisation/affiliation, Journal/venue, Project, Keywords, Identifier (grouped with dropdown-header labels).

Picker — selected / applied items

When a filter is already applied (or is the one currently being edited), its picker item is marked with Bootstrap's .active state plus a trailing check icon. Reopening the picker then shows at a glance which dimensions are in play — without this, an applied filter looks identical to an unused one. Driven by aria-current="true" for assistive tech.

Selected / applied items

The .active fill is recoloured via --bs-dropdown-link-active-bg: var(--bt-blue-200) and --bs-dropdown-link-active-color: rgba(var(--bs-dark-rgb), 1) (text-dark) — a global --bs-dropdown-* override in foundation/_bootstrap-overrides.scss.

filter-bar.js toggles this from the active-filter state on every apply / remove / clear, and while a filter is being edited. Picking an item closes the picker and opens the editor, so the highlight is what you see when you reopen Add filter.

Editor — checklist

Used for enumerable values (Faculty or department). Multiple selections produce a chip like "Sciences +2". The .bt-panel__body--checklist container scrolls when the list is long.

Editor as overlay

The editor panel floats over the result list instead of pushing it down. The filter bar and panel are wrapped in a position-relative container; the panel (#wf-filter-editor / #rdir-filter-editor / #pdir-filter-editor) then gets position-absolute top-100 start-0 w-100 mt-2 — anchored to the bottom edge of the bar, full width, with a small gap.

The popover z-index (and a min-width floor for narrow anchors) lives on .bt-panel itself — Bootstrap's z-3 utility maxes at 3, which ties with the active .page-link in pagination.

Active chips

Applied filters surface as chips — the visible output of the picker/editor flow. The standard is the editable split chip: the label reopens the editor, the × removes it — two clickable badges (outline, squared) joined with .filter-chip-group. Display-only summaries use a soft badge text-bg-primary-light.

Wiring reference

Each bar prefixes its IDs — wf- for works, rdir- for researchers, pdir- for projects — all driven by filter-bar.js. The set below is shown with the rdir- prefix; the engine exits silently for a bar if any are missing.

ID / attribute Element Purpose
id="rdir-add-filter-dropdown" Dropdown wrapper div Outside-click close handler
id="rdir-add-filter-btn" Bootstrap dropdown trigger button Closed programmatically when an item is selected
id="rdir-filter-picker-list" .bt-dropdown-scroll Contains the [data-filter] items
data-filter="<key>" Each button.dropdown-item Key must match an entry in FILTERS inside the directory engine
id="rdir-filter-editor" div.bt-panel.bt-panel--wide (hidden by default) Editor panel — the engine renders into this and shows/hides it
id="rdir-active-chips" div[role="group"] Chip output — the engine renders and manages chips here
id="rdir-clear-all" button (hidden by default) Shown when chips are active; resets all filters on click
