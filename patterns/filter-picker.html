Filter picker & editor

One connected interaction: a grouped filter menu (the picker) and an inline editor panel that opens when a filter is chosen. Together they replace the traditional sidebar-only facet approach with something that scales to any number of filter dimensions without adding layout complexity.

How it works

The full flow, end to end:

  1. User clicks Add filter → Bootstrap dropdown opens showing the picker
  2. User clicks a filter item (e.g. Author) → dropdown closes, editor panel appears inline below the search bar
  3. User sets a value → clicks Apply → editor closes, a chip appears in the active filter bar
  4. User clicks a chip → editor reopens pre-populated for editing
  5. User clicks × on a chip → filter removed

All wired by filter-editor.js. The people-search editor type also requires people-search.js and (in prototypes) people-search-stub.js.

Picker — backoffice

Used in backoffice-search.html and filter-editor.html. Includes a search input that filters the list live and hides empty groups. The scrollable .bt-filter-picker__list keeps the dropdown bounded regardless of list length.

Editor — checklist

Used for enumerable values: Type, Subtype, Status, Publication status. Multiple selections produce a chip like "Journal article +2". The .filter-checklist container scrolls when the list is long.

Editor — boolean

Active chips

Each applied filter becomes a split chip. The left part (editable) reopens the editor. The × removes the filter. Rendered by filter-editor.js into #active-chips. See Filter tags for the chip component in isolation.

Wiring reference

Every page using this pattern must include these IDs. filter-editor.js exits silently if any are missing.

ID / attribute Element Purpose
id="add-filter-dropdown" Dropdown wrapper div Outside-click close handler in filter-editor.js
id="add-filter-btn" Bootstrap dropdown trigger button Closed programmatically when an item is selected
id="filter-picker-list" .bt-filter-picker__list Targeted by the filter search handler; contains [data-filter] items
data-filter="<key>" Each button.dropdown-item Key must match an entry in FILTERS inside filter-editor.js
id="filter-search" input[type="search"] inside picker Optional. Live-filters the list when present
id="filter-editor" div.filter-editor (hidden by default) Editor panel — filter-editor.js renders into this and shows/hides it
id="active-chips" div[role="group"] Chip output — filter-editor.js renders and manages chips here
id="clear-all-btn" button (hidden by default) Shown when chips are active; resets all filters on click
