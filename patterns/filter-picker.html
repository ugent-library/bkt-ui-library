Filter picker & editor

One connected interaction: a grouped filter menu (the picker) and an inline editor panel that opens when a filter is chosen. It complements the sidebar facets — handling high-cardinality dimensions (author, venue, project) that don't fit a fixed checklist — and scales to any number of filter dimensions without adding layout complexity.

How it works

The full flow, end to end:

  1. User clicks Add filter → Bootstrap dropdown opens showing the picker
  2. User clicks a filter item (e.g. Author) → dropdown closes, editor panel appears inline below the search bar
  3. User sets a value → clicks Apply → editor closes, a chip appears in the active filter bar
  4. User clicks a chip → editor reopens pre-populated for editing
  5. User clicks × on a chip → filter removed

Wired by filter-editor.js (the search-filter-bar.html partial), or by the directory-filters*.js variants on the public directory result bars (result-filter-bar*.html). The people-search editor type also requires people-search.js and (in prototypes) people-search-stub.js.

Picker — backoffice

The full filter set with a search input — the search-filter-bar.html partial driven by filter-editor.js. Intended for the backoffice and Advanced search surfaces; not yet wired into a live page. The search input filters the list live and hides empty groups, and the scrollable .bt-dropdown-scroll keeps the dropdown bounded regardless of list length.

Picker — selected / applied items

When a filter is already applied (or is the one currently being edited), its picker item is marked with Bootstrap's .active state plus a trailing check icon. Reopening the picker then shows at a glance which dimensions are in play — without this, an applied filter looks identical to an unused one.

The .active fill is recoloured to the theme chip blue with dark text (rather than Bootstrap's solid primary) so it reads as a soft selection consistent with the active chips. Add d-flex align-items-center to the item so the check sits at the trailing edge via ms-auto. Driven by aria-current="true" for assistive tech.

Editor as overlay

By default the editor panel sits inline and pushes the content below it down when it opens. In dense result views (e.g. the entity result filter bar) that shift is jarring. Wrap the bar and editor in a position-relative container and give the panel Bootstrap's position utilities so it floats over the result list instead.

Wrap the bar and panel in a position-relative container, then give the #filter-editor panel position-absolute top-100 start-0 w-100 mt-2: it anchors to the bottom edge of the bar at full width, above the results, with a small gap. The popover z-index (and a min-width floor for narrow anchors) lives on .bt-panel itself — Bootstrap's z-3 utility maxes at 3, which ties with the active .page-link in pagination.

Active chips

Applied filters surface as chips — the visible output of the picker/editor flow. The directory result bars render the editable split chip (the label reopens the editor, the × removes it); public-works.html uses the basic single-button chip (the whole chip removes the filter). Both are clickable badges (outline, squared); the split chip joins two of them with .filter-chip-group. Display-only summaries use a soft badge text-bg-primary-light. This pattern just composes them.

Wiring reference

Every page using this pattern must include these IDs. filter-editor.js exits silently if any are missing.

ID / attribute Element Purpose
id="add-filter-dropdown" Dropdown wrapper div Outside-click close handler in filter-editor.js
id="add-filter-btn" Bootstrap dropdown trigger button Closed programmatically when an item is selected
id="filter-picker-list" .bt-dropdown-scroll Targeted by the filter search handler; contains [data-filter] items
data-filter="<key>" Each button.dropdown-item Key must match an entry in FILTERS inside filter-editor.js
id="filter-search" input[type="search"] inside picker Optional. Live-filters the list when present
id="filter-editor" div.bt-panel.bt-panel--wide (hidden by default) Editor panel — filter-editor.js renders into this and shows/hides it
id="active-chips" div[role="group"] Chip output — filter-editor.js renders and manages chips here
id="clear-all-btn" button (hidden by default) Shown when chips are active; resets all filters on click
