Templates

Public and backoffice prototypes

Full-page reference implementations, grouped by application (public site, researcher, team). Start here to see patterns assembled into a real page. Usable for user testing.

Some pages under templates ship more than one state — for example filled, empty, or message — so you can see how a screen behaves with and without data. When you open such a template, small state buttons appear beneath its name in the sidebar. Click one to switch, or press Ctrl+E to cycle through them. The plain page address always shows the first state.
