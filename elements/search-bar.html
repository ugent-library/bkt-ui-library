Search bar

A pill-shaped search field with an inset submit button. Built as a modifier on Bootstrap's .input-group — Bootstrap handles the flex layout, border joining, and sizing scale; the modifier adds the pill shape, the whole-bar focus ring, and search-cancel suppression.

Default

Default — .input-group.input-group--search

html
<form role="search" class="input-group input-group--search"
  aria-label="Search research output" action="/search">
  <label class="visually-hidden" for="q">Search research output</label>
  <input class="form-control" type="search" id="q" name="q"
    placeholder="Search by title, author, or DOI…" autocomplete="off">
  <button class="btn btn-primary" type="submit">Search</button>
</form>

What the modifier does — and what it leaves alone

Bootstrap's .input-group provides: flex layout, border joining between input and button, and the .input-group-lg / .input-group-sm sizing scale.

.input-group--search adds only three things on top: the pill border-radius on the outer wrapper and the input's leading edge; a :focus-within ring on the whole bar as one unit (individual element focus shadows are suppressed); and the native browser search-cancel button is hidden.

Accessibility notes

  • Wrap in <form role="search"> with a distinct aria-label so screen reader users can jump to the search landmark directly.
  • The <label> must exist in the DOM even if visually hidden. placeholder alone is not a label.
  • Use type="search" so mobile browsers show the correct keyboard and voice-search affordances.
  • The :focus-within ring surrounds the entire pill — individual element outlines are suppressed to avoid a double ring. The ring meets WCAG 2.2 AA contrast requirements.
  • The button must be type="submit" so the form submits without JavaScript. action on the <form> is the no-JS fallback.
  • When wiring to HTMX live search, add hx-get on the input and a paired hx-indicator with aria-live="polite".

When to use this vs the toolbar search

Use .input-group--search when search is the primary action on the page and deserves visual prominence — homepage hero, a dedicated search landing page, or above a full-width result list.

Use the compact .form-control-search inside a .bt-toolbar for inline filtering in the backoffice — filtering a list, searching within a deposit, live-filtering a dropdown. That pattern is documented in Forms & Inputs → Search input.
