Alerts

Alerts communicate system state. They are never decorative. Each variant maps to a specific type of information — using the wrong variant erodes trust. The system tells you what happened and why. Never leave the user wondering.

Writing alerts

✓ Do

Say what happened: "Deposit submitted successfully"
Say what to do next: "You will receive an email when published"
Be specific about errors: "File exceeds 500 MB limit"
Blame the system, not the user

✗ Don't

Vague: "An error occurred"
Blaming: "You uploaded the wrong file type"
Using danger for non-critical info
Showing alerts that auto-dismiss before they're read

Accessibility

  • Always include role="alert" — this triggers screen reader announcement
  • For non-urgent messages, use role="status" instead
  • Don't rely on colour alone — icons and text carry the meaning
  • Dismissible alerts must have a visible close button with aria-label="Close"
  • Dynamically injected alerts (HTMX) must be inside an aria-live region
html — live region for HTMX alerts
<!-- Place once near top of page, inject alerts into it with HTMX -->
<div aria-live="polite" aria-atomic="true" id="alert-region"></div>

<!-- HTMX response targets this region -->
<form hx-post="/deposit" hx-target="#alert-region" hx-swap="innerHTML">
  ...
</form>
