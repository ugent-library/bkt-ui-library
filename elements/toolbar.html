Accessibility

  • Add role="search" to search toolbars with aria-label="Search and filter research output"
  • Use semantic headings (<h1>) in bt-toolbar__title when appropriate
  • bt-toolbar__item elements are flex items within bt-toolbar__left/bt-toolbar__right/bt-toolbar__middle containers
  • Navigation toolbars should use <nav> in the middle section
  • Screen readers navigate toolbar content as part of the page flow
  • Interactive elements within toolbars remain fully keyboard accessible

Implementation notes

  • Use bt-toolbar for the bar layout itself and Bootstrap controls inside it.
  • For card/table result switches, use a Bootstrap btn-group with input.btn-check and matching label.btn controls.
  • The old standalone view-toggle documentation page was removed because Bootstrap already provides the correct control mechanics.
