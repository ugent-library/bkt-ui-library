Buttons

Five variants, three sizes. Primary is a commitment — one per view. Ghost for reversible secondary actions. Danger only for irreversible consequences. The interface has already made most decisions; buttons confirm the ones that are genuinely the user's to make.

Variants

Primary — one per view, confirms the main action

Secondary — outlined, supports the primary

Ghost — low-emphasis, cancel or navigate back

Danger — only for irreversible destructive actions

On dark surfaces — btn-primary and btn-outline-white

Link — inline action that reads as text

Sizes

Large — btn-lg — 44px — confirmation dialogs, hero CTAs

Default — 38px — standard toolbar and form actions

Small — btn-sm — 32px — inline row actions, compact toolbars

Loading state

After a click, the button replaces its label with a spinner and becomes aria-disabled. The spinner is Bootstrap's spinner-border-sm — no custom classes. In HTMX, the spinner lives inside .htmx-indicator and is shown automatically during the request; the static demo below shows what each state looks like.

Usage rules

✓ Do

  • One primary button per view
  • Use ghost for cancel / go back
  • Use danger only when the action cannot be undone
  • Button labels are verbs: Download, Save, Delete
  • Icon-only buttons carry an aria-label that includes the record name
  • Prefer aria-disabled + tooltip over hiding unavailable actions

✗ Don't

  • Two primary buttons side by side
  • Use danger for recoverable deletes
  • Vague labels: Click here, OK, Submit
  • Silently disable without explaining why
  • Put aria-label on a button that already has visible text
  • Use btn-link for navigation — use <a> instead
