Buttons

Four variants, three sizes. Primary is a commitment — one per view. Ghost for reversible secondary actions. Danger only for irreversible consequences. The interface has already made most decisions; buttons confirm the ones that are genuinely the user's to make.

Usage rules

✓ Do

  • One primary button per view
  • Use ghost for cancel / go back
  • Use danger only when action cannot be undone
  • Button labels are verbs: Download, Save, Submit

✗ Don't

  • Two primary buttons side by side
  • Use danger for "Delete" if it's recoverable
  • Vague labels: Click here, OK, Submit
  • Disabled buttons without explanation
