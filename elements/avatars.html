Avatars

Avatars display initials in a round chip. They appear as the account-menu trigger in the backoffice topbar and in author and contributor rows. When an avatar is a static label its name lives in adjacent text; when it is the interactive control, the accessible name goes on the control.

Accessibility

  • When the avatar is a static label and the name is in adjacent text, mark it aria-hidden="true" so the initials are not announced twice
  • When the avatar is the interactive control (account-menu trigger), put the accessible name on the control with aria-label
  • An icon inside an avatar is decorative — give it aria-hidden="true"
  • A faculty SVG glyph carries the faculty identity: give the img a real alt when the chip stands alone, or alt="" when the faculty name is already in adjacent text
  • Initials should be 2 characters maximum for readability
  • Colour is never the only signal — a coloured chip still carries initials or an icon
