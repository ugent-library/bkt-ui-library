View Toggle

Toggle buttons for switching between card and table views. Uses aria-pressed for state indication. Groups related toggle buttons with role="group".

JavaScript integration

Toggle behavior
javascript
// Handle view toggle clicks
document.querySelectorAll('.view-toggle-btn').forEach(btn => {
  btn.addEventListener('click', function() {
    // Remove active state from all buttons in group
    this.parentElement.querySelectorAll('.view-toggle-btn').forEach(sibling => {
      sibling.classList.remove('is-active');
      sibling.setAttribute('aria-pressed', 'false');
    });

    // Add active state to clicked button
    this.classList.add('is-active');
    this.setAttribute('aria-pressed', 'true');

    // Switch view (implement your view switching logic)
    const viewType = this.getAttribute('aria-label').toLowerCase();
    switchView(viewType); // e.g., 'card view' or 'table view'
  });
});

Accessibility

  • Wrap in role="group" with descriptive aria-label
  • Use aria-pressed for toggle state (not aria-checked or role="tab")
  • Each button needs an aria-label describing the view it activates
  • Icons are decorative (aria-hidden="true")
  • Active state is indicated by is-active class and aria-pressed="true"
  • Screen readers announce the pressed state and button purpose
