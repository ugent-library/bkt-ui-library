Foundations

Utilities

Bootstrap is the primary utility system. Booktower keeps only a very small helper layer for a few repeated semantic tokens used across the prototypes.

Rule Of Thumb

Use Bootstrap for

  • spacing and gaps
  • flex and grid layout
  • sizing and widths
  • text alignment and display
  • standard `border`, `rounded`, `text-*`, `bg-*` helpers where Bootstrap already expresses the intent

Use Booktower utilities for

  • the app/body surface background
  • the dark surface background
  • the white card background when it maps to the token
  • the shared design-system border token

Supported Booktower Utilities

These are the only custom utilities that should be relied on in page markup.

Backgrounds

.bt-bg

Uses --bt-bg. Good for page sections and neutral app surfaces.

.bt-bg-alt

Uses --bt-bg-alt. Good for subtle chrome and secondary blocks.

.bt-bg-dark

Uses --bt-bg-dark. Good for dark token-backed avatars and dark panels.

.bt-bg-white

Uses --bt-white. Good when a token-backed white surface is needed explicitly.

Borders

.bt-border

Uses the shared token-backed border value --bt-border.

Bootstrap Color, Text, And Border Utilities

These are the Bootstrap utilities you should reach for first. In this library they inherit Booktower tokens where Bootstrap has been overridden, so they stay visually aligned with the system.

Text Utilities

.text-primary

.text-success

.text-warning

.text-danger

.text-muted

.text-body

.text-black

Use these for state text, secondary copy, and standard semantic emphasis. Do not create `bt-text-*` equivalents.

Background Utilities

.bg-primary
.bg-success
.bg-warning
.bg-danger
.bg-secondary
.bg-white

These are especially appropriate for badges, alerts, and small semantic surfaces. For page chrome and shell backgrounds, prefer the smaller Booktower helper set above.

Subtle Background Utilities

.bg-primary-subtle
.bg-success-subtle
.bg-warning-subtle
.bg-danger-subtle

Bootstrap's built-in light tints, backed by --bs-*-bg-subtle (which inherit Booktower's theme colours). Use them for soft status surfaces — alert backgrounds, highlighted rows, tinted callouts. Pair with the matching .text-*-emphasis for readable foreground text. secondary, info, light and dark subtle variants also exist.

Border Utilities

.border
.border-top .border-bottom
.border-start
.border-end

Prefer Bootstrap directional border helpers over custom `bt-border-*` utilities. Keep `bt-border` only for the shared token-backed shorthand.

Faculty Colors

Solid utilities

Use .bg-faculty-* for a solid faculty chip or block. Each sets the background and a readable text/icon colour for that faculty (dark ink where the brand colour is too light for white). Used by bt-avatar faculty chips and the organisation directory.

Light utilities

.bg-faculty-*-light is a 12% tint of the brand colour, computed so it always tracks the token. Safe to hold body text (--bt-text clears 4.5:1 on every faculty); use --bt-text, not --bt-text-muted. For tinted sections, callouts and cards keyed to a faculty — not avatar chips. Relies on CSS color-mix.

What Does Not Exist

Do not assume there is a Booktower utility for every token. These are intentionally not part of the runtime utility surface:

  • bt-text-blue, bt-text-green, bt-text-danger
  • bt-bg-blue, bt-bg-success, bt-bg-warning
  • bt-border-top, bt-border-right, bt-border-bottom, bt-border-left
  • one-off width, spacing, display, or alignment helpers
