Accessibility statement

Ghent University Library is committed to making its websites accessible, in accordance with the Bestuursdecreet of 7 December 2018, which transposes European Directive 2016/2102 (opens in new window).

This accessibility statement applies to lib.ugent.be — the website of the Ghent University Library (Dutch and English versions).

About this website

lib.ugent.be consists of two parts with different technical ownership:

Content pages (lib.ugent.be)
Library information, services, opening hours, and contact pages. Built and maintained by Ghent University and Ghent University Library.
Search and catalogue (libcatalog.ugent.be)
The homepage, search functionality, and catalogue are provided by Ex Libris Primo NDE (opens in new window), a third-party discovery platform. We can configure some aspects (labels, colours, page titles) and apply CSS overrides, but we cannot change the underlying application code. Issues in the core application require action by the vendor.

Compliance status

This website partially complies with the Web Content Accessibility Guidelines (WCAG) 2.1 (opens in new window) level AA. The non-compliant areas are listed below.

Non-accessible content

The content listed below is not yet accessible for the following reasons.

Content pages (lib.ugent.be)

WCAG criterion Issue Severity
2.4.1 Bypass blocks There is no skip link on content pages. Keyboard users must tab through the entire navigation to reach the main content. High
2.4.7 Focus visible The focus indicator is suppressed on all interactive elements (outline: none with no visible replacement). Keyboard users cannot see which element is currently focused. High
1.3.1 Info and relationships Some pages have heading hierarchy issues. For example, an <h3> follows directly after <h1>, skipping the <h2> level. On another page, an empty heading is present in the content. Moderate
1.3.1 Info and relationships The <main> element has no id attribute, so a skip link would have no target. Moderate
1.3.1 Info and relationships Breadcrumb navigation uses <ul> instead of <ol>. Position in the hierarchy is meaningful and should use an ordered list. Low
1.4.3 Contrast (minimum) The cookie consent button has white text on a blue background with a contrast ratio of 4.27:1, below the required 4.5:1 minimum for normal-sized text. Low
2.4.4 Link purpose (in context) The footer logo link to ugent.be opens in a new window without a visible or programmatic warning. Low

Search and catalogue (Ex Libris Primo NDE)

The following issues were found in the Primo NDE application. We have noted which issues are within our configuration control and which require action by the vendor.

WCAG criterion Issue Severity Can we fix this?
WCAG 2.1, Success Criteria, 1.4.10 (AA) Content can be enlarged without increasing line length. Requires 400% zoom to work. High No — not configurable in Primo NDE nor fixable within reasonable effort, without rewriting the entire application css. Raised with vendor.
2.4.7 Focus visible Focus outline is suppressed on many elements. Some elements use box-shadow on :focus-visible but the coverage is inconsistent — not all focusable elements receive a visible indicator. High Partially — we can apply CSS overrides for some elements
1.3.1 Info and relationships The <h1> text is "home" or "search" — not meaningful page headings. High No — not configurable in Primo NDE. Raised with vendor.
1.3.1 Info and relationships An empty <h2> is present in the DOM. Moderate No — vendor issue
1.3.1 Info and relationships Heading hierarchy is broken: search results heading is <h4> after a preceding <h2>, skipping <h3>. Moderate No — vendor issue
1.3.1 Info and relationships Four of five <nav> elements have no aria-label. Screen reader users cannot distinguish between them. Moderate No — vendor issue
4.1.2 Name, role, value Several <mat-icon role="img"> elements have no accessible name. Moderate No — vendor issue
2.4.2 Page titled The English homepage title still shows the Dutch name "Universiteitsbibliotheek Gent" instead of an English title. Low Investigating.
2.4.4 Link purpose (in context) External links open in a new window without visible warning text. Low Investigating — may be configurable via CSS/labels

What we have done

We have reported the vendor-side issues listed above to Ex Libris. Where possible, we apply CSS overrides and configuration changes to improve accessibility within the constraints of the platform.

Planned improvements

Medium term (3–6 months)

Address the contrast issue on the cookie consent widget. Investigate empty CMS headings. Investigate remaining Primo NDE configuration options for page titles and link text.

How we tested

This statement is based on two accessibility audits of lib.ugent.be.

First audit October 2025 — manual audit by Ghent University Library
Second audit April 2026 — manual and automated audit by Ghent University Library
Scope Dutch and English versions of lib.ugent.be (content pages) and libcatalog.ugent.be (Primo NDE homepage and search)
Standard WCAG 2.1 level AA (EN 301 549)
Automated tools WAVE
Manual testing Visual inspection (Safari, Chrome, Firefox, iOS), keyboard navigation (Safari, Chrome, Firefox), screen reader (VoiceOver), DOM and computed style inspection
Limitations Search result pages could not be fully tested in an unauthenticated session. Facet checkbox labelling, modal keyboard trapping, and form validation in the Primo interface were not covered. A follow-up audit with an authenticated session is planned.
Pages tested
  • lib.ugent.be/nl and /en (homepage — redirects to Primo NDE)
  • lib.ugent.be/nl/libraries (+ individual library pages including Boekentoren)
  • lib.ugent.be/nl/services (+ subpages: registration, loans, scanning, ILL, study locations, etc.)
  • lib.ugent.be/nl/info (+ subpages: regulations, research tips, e-resources, open data, etc.)
  • libcatalog.ugent.be search results page (limited — guest session)
