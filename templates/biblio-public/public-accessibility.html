Accessibility statement

Ghent University Library is committed to making its websites accessible, in accordance with the Bestuursdecreet of 7 December 2018, which transposes European Directive 2016/2102 (opens in new window).

This accessibility statement applies to lib.ugent.be — the website of the Ghent University Library (Dutch and English versions).

About this website

lib.ugent.be consists of two parts with different technical ownership:

Content pages (lib.ugent.be)
Library information, services, opening hours, and contact pages. Built and maintained by Ghent University and Ghent University Library.
Search and catalogue (libcatalog.ugent.be)
The homepage, search functionality, and catalogue are provided by Ex Libris Primo NDE (opens in new window), a third-party discovery platform. We can configure some aspects (labels, colours, page titles) and apply CSS overrides, but we cannot change the underlying application code. Issues in the core application require action by the vendor.

Compliance status

This website partially complies with the Web Content Accessibility Guidelines (WCAG) 2.2 (opens in new window) level AA. The non-compliant areas are listed below.

Non-accessible content

The content listed below is not yet accessible for the following reasons.

WCAG criterion Issue Severity

Search and catalogue (Ex Libris Primo NDE)

The following issues were found in the Primo NDE application. We have noted which issues are within our configuration control and which require action by the vendor.

WCAG criterion Issue Severity Can we fix this?
WCAG 2.2, Success Criteria, 1.4.10 (AA) Content can be enlarged without increasing line length. Requires 400% zoom to work. High No — not configurable in Primo NDE nor fixable within reasonable effort, without rewriting the entire application css. Raised with vendor in November 2025, fix planned in July 2026. Users needing high zoom levels are advised to use the browser's simplified view or contact us for assistance in retrieving specific metadata.
1.3.1 Info and relationships The <h1> text is "home" or "search" — not meaningful page headings. High No — not configurable in Primo NDE. Raised with vendor in April 2026.
1.3.1 Info and relationships An empty <h2> is present in the DOM. Low No — vendor issue. Raised with vendor in April 2026.
1.3.1 Info and relationships Heading hierarchy is broken: search results heading is <h4> after a preceding <h2>, skipping <h3>. Moderate No — vendor issue. Will raise this when other problems have been resolved.
1.3.1 Info and relationships Some <nav> elements have no aria-label. Screen reader users cannot distinguish between them. Moderate No — vendor issue. Will raise this when other problems have been resolved.
2.4.2 Page title The English homepage title still shows the Dutch name "Universiteitsbibliotheek Gent" instead of an English title. Low Investigating configuration.
2.4.4 Link purpose (in context) Links in the search results list (such as "Available in" or "Details") lack a descriptive ARIA label containing the title of the work. Moderate No — Vendor issue. Raised with vendor in April 2026.

What we have done

We have reported the vendor-side issues listed above to Ex Libris. Where possible and proportionate, we apply CSS overrides and configuration changes to improve accessibility within the constraints of the platform.

Resolved in April 2026 (verified on live site)

  • 2.4.1 Skip link: now present for faster keyboard navigation on lib.ugent.be
  • 1.3.1
    id: now has id="main-content" for skip-link target
  • We have improved the focus visibility for interactive elements on css-overrides.
  • 1.3.1 Heading hierarchy: alert component now uses h2, empty CMS heading on /nl/services/ill and /en/services/ill removed
  • 1.3.1 Breadcrumb: now uses "ol" instead of "ul"
  • 2.4.4 Footer new-window link: now has an indicator that it opens in a new window
  • 1.3.1 Info and relationships: We have done a systematic review. We have added unique aria labels to navigation regions across both domains (
    elements). Not yet for all libcatalog.ugent.be navigation (Ex Libris).

Planned improvements

Short term (3 months)

Investigate remaining Primo NDE configuration: improve page title. Address the contrast issue on the cookie consent widget.

Medium term (3–6 months)

Follow up on vendor-side issues with Ex Libris and verify fixes on the live site. Investigate solutions if no vendor fixes are possible. Follow up on nav issue and headings issue with Ex Libris, when there is progress on the other ones to avoid overload.

How we tested

This statement is based on two accessibility audits of lib.ugent.be.

First audit October 2025 — manual audit by Ghent University Library
Second audit April 2026 — manual and automated audit by Ghent University Library
Scope The audit covered the online service of Ghent University Library, in which lib.ugent.be (information pages) and libcatalog.ugent.be (search engine/catalogue) were tested as a single integrated user experience. Although technically divided across various subdomains, the navigation flow from landing to detail page was assessed against WCAG guidelines as a whole.
Standard WCAG 2.2 level AA (EN 301 549)
Automated tools WAVE
Manual testing Visual inspection (Safari, Chrome, Firefox, iOS), keyboard navigation (Safari, Chrome, Firefox), screen reader (VoiceOver), DOM and computed style inspection
Limitations Search result pages could not be fully tested in an unauthenticated session. Facet checkbox labelling, modal keyboard trapping, and form validation in the Primo interface were not covered. A follow-up audit with an authenticated session is planned.
Pages tested
  • lib.ugent.be/nl and /en (homepage — redirects to Primo NDE)
  • lib.ugent.be/nl/libraries and /en/libraries (+ individual library pages including Boekentoren)
  • lib.ugent.be/nl/services and /en/services (+ subpages: registration, loans, scanning, ILL, study locations, etc.)
  • lib.ugent.be/nl/info and /en/info (+ subpages: regulations, research tips, e-resources, open data, etc.)
  • libcatalog.ugent.be search results page
  • libcatalog.ugent.be detail pages
